# Engineering Archive

Use these active files first:

- `README.md` for current release state, setup, bug-fix handoff, and release flow.
- `docs/engineering/AGENT_RUNBOOK.md` for build/release traps and validation commands.
- `docs/engineering/FEOXDB_SYNC_MIGRATION_PLAN.md` for current shared-sync architecture.
- `docs/engineering/SYNC_RECOVERY_INVARIANTS.md` for recovery rules.
- `docs/engineering/PROJECT_FOLDER_BREAKDOWN.md` for the tracked-file map.

Archived files:

- `CLEANUP_CHECKLIST.md`: old release evidence and version-by-version staging records.
- `CODE_BEHAVIOR_AUDIT.md`: source audit notes that started the hardening pass.
- `CODE_BEHAVIOR_REMEDIATION_CHECKLIST.md`: old hardening checklist and release gates.
- `DONE_CHECKLIST.md`: completed cleanup work, worker logs, and old release notes.
